# Git Workflow

## Recommended branches

- `main`: stable MVP for thesis presentation
- `develop`: integration branch
- `feature/*`: focused improvements

## Suggested commit style

- `feat:` new functionality
- `docs:` documentation updates
- `refactor:` structural improvements
- `test:` test additions

## Suggested first sequence

1. run MVP locally,
2. inspect `output/`,
3. commit generated repository structure,
4. iterate by module,
5. introduce uploader/replayer in later milestones.
